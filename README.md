# VEP Classifier

[![DOI](https://zenodo.org/badge/1144714879.svg)](https://zenodo.org/badge/latestdoi/1144714879)

A machine-learning pipeline for classifying **visually evoked potentials (VEPs)** recorded from the visual cortex of 
rats implanted with a retinal prosthesis. 
The pipeline preprocesses raw electrophysiology recordings (CSV files), 
trains a 1D CNN classifier, and produces binary predictions regarding which cells contributed to the evoked response through a Flask web 
application.

**Classification targets** – the model distinguishes between the following two labels:

| Label        | Meaning                                                                                    |
|--------------|--------------------------------------------------------------------------------------------|
| `BC_Only`    | Response driven by bipolar cells only (network response)                                   |
| `BC_and_RGC` | Mixed response from bipolar cells and retinal ganglion cells (network and direct response) |

## Installation

```bash
# 1. Create and activate a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate   # macOS / Linux
# .venv\Scripts\activate    # Windows

# 2. Install dependencies and the project itself (editable)
pip install -r requirements.txt
pip install -e .
```

> `pip install -e .` registers the `core`, `classifiers`, `experiments`, and `app`
> packages so that qualified imports (e.g. `from classifiers.CNN_classifier import CNN1D`)
> resolve from any working directory — including inside notebooks.

> **TensorFlow note:** `tensorflow` on Apple Silicon requires the `tensorflow-macos` / `tensorflow-metal` packages instead. See the [TF macOS guide](https://developer.apple.com/metal/tensorflow-plugin/) for details.

## Data layout

Raw labelled recordings must follow this directory structure before any preprocessing:

```
data/Labelled_VEP_Data/
└── <DEVICE>/          # e.g. PRIMA_LE_DA, MP20_LE_DA, ...
    ├── BC_Only/       # one CSV per trial
    └── BC_and_RGC/
```

Each raw CSV must have the columns `Step 1` (time in ms), `Chan 1`, and `Chan 3` (signal amplitudes), with a header row and one sub-header row that the loader skips automatically. `Chan 1` is used for baseline alignment and optional artifact removal; `Chan 3` is the classified signal.

## Preprocessing

`core/preprocessing.py` converts raw recordings to standardised waveforms and saves them to `data/Preprocessed_VEP_Data/`. Run it from a notebook or script:

```python
from core.preprocessing import preprocess_save_all

preprocess_save_all(
    base_dir="data/Labelled_VEP_Data",
    output_dir="data/Preprocessed_VEP_Data",
    normalize=True,
    do_artifact_removal=False,
    do_dwt_downsampling=True,
    tmax=400,           # trim to 0–400 ms post-stimulus
    SNR_filtering=True,
    SNR_threshold=1.0,  # discard trials with SNR < 1
)
```

The pipeline per trial:
1. Load raw CSV → extract `Chan 1` and `Chan 3`
2. Optional artifact removal (scale and subtract `Chan 1` over the stimulation period)
3. Average the two stimulus phases
4. Compute SNR; discard trial if below threshold
5. Trim to `[0, tmax]` ms
6. DWT downsampling (`db4` wavelet, level 4 by default)
7. Min-max normalisation

## Training a new model

Open and run **`app/train_and_save_full_model.ipynb`**.

The notebook:
1. Sets the working directory to the repo root.
2. Loads preprocessed data for the chosen device via `core.helpers.get_data`.
3. Instantiates a `CNN1D` classifier and calls `fit_and_save`, which trains on the **full dataset** (no validation split) and writes three artefacts to `app/saved_models/`:
   - `<prefix>_<timestamp>.keras` – Keras model weights
   - `<prefix>_<timestamp>_scaler.pkl` – fitted `StandardScaler`
   - `<prefix>_<timestamp>_metadata.json` – class labels, hyperparameters, preprocessing parameters

```python
from classifiers.CNN_classifier import CNN1D
from core.helpers import get_data

X, y, _, file_list = get_data(device="PRIMA_LE_DA", classes=2)

cnn = CNN1D(learning_rate=5e-4, dropout_rate=0.5, l2_lambda=1e-3)
paths = cnn.fit_and_save(
    X, y,
    epochs=50,
    batch_size=16,
    save_dir="app/saved_models",
    model_name="prima_le_da_2class",
    device="PRIMA_LE_DA",
    preprocessing_info={
        "normalize": True,
        "artifact_removal": False,
        "dwt_downsampling": True,
        "tmax": 400,
        "dwt_level": 4,
    },
)
print(paths)  # {'model_path': ..., 'scaler_path': ..., 'metadata_path': ...}
```

The web app automatically loads the **most recent** artefacts from `app/saved_models/`, so
no manual path edits are needed after training.

## Running the web app

```bash
python -m app.app
```

The app starts on **http://localhost:8080**. Open that URL in a browser to access the drag-and-drop interface.

### What the app does

1. Accepts a single raw VEP CSV file (same format as `data/Labelled_VEP_Data/`; max 16 MB).
2. Runs the full preprocessing pipeline stored in the model's metadata JSON.
3. Returns:
   - Predicted class (one of the classes in the loaded model's metadata — `BC_Only` / `BC_and_RGC` for the default 2-class model, or all three when a 3-class model is loaded)
   - Confidence (max softmax probability)
   - SNR of the uploaded signal
   - Per-class probability breakdown
   - An inline plot of the preprocessed waveform

#### Health check

```bash
curl http://localhost:8080/health
```

Returns a JSON summary of the loaded model and its classes.
